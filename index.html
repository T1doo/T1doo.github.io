实验1：MIPS程序设计

1. CISC 与 RISC

CISC（Complex Instruction Set Computer） RISC（Reduced Instruction Set Computer）
指令系统 复杂，指令数量多、功能强 精简，只包含最常用指令
指令长度 可变，格式多样 固定长度
寻址方式 多种 较少
内存访问 多种指令可访问内存 只有 Load/Store 能访问内存
寄存器 较少 提供大量通用寄存器
代表 VAX MIPS

例：同一操作在 CISC 与 RISC 下的对比▶

VAX (CISC)：一条指令完成内存到内存的加法 

> ADDL (R9), (R10), (R11)   ; mem[R9] ← mem[R10] + mem[R11]
>
MIPS (RISC)：需要拆分为 Load → 运算 → Store ⭐ 
> lw  R1, (R10)     # R1 ← mem[R10]
> lw  R2, (R11)     # R2 ← mem[R11]
> add R3, R1, R2    # R3 ← R1 + R2
> sw  R3, (R9)      # mem[R9] ← R3
>

2. MIPS 体系结构设计准则

  1. 简单设计有助于规整化 — 指令长度相同，格式固定
  2. 加快常见功能 — 指令集中只包含最常用的指令
  3. 越小的设计越快 — 较少的硬件将有更少的延迟
  4. 好的设计需要好的折中方法 — （既要固定指令长度又要支持多种指令），指令有 3 种格式，采用扩展码技术

A simpler CPU is a faster CPU. — John Hennessy

3. MIPS 操作数

3.1 寄存器

MIPS 有 32 个 32 位通用寄存器：

名称 编号 用途 名称 编号 用途
$0 0 常数 0 $t8~$t9 24~25 临时变量
$at 1 汇编器临时变量 $k0~$k1 26~27 OS 临时变量
$v0~$v1 2~3 函数返回值 $gp 28 全局指针
$a0~$a3 4~7 函数参数 $sp 29 栈指针
$t0~$t7 8~15 临时变量 $fp 30 帧指针
$s0~$s7 16~23 保存变量 $ra 31 函数返回地址
也就是，程序里面的变量就是编号8~25的部分

指令基本结构

$$\text{指令} = \text{操作 (operation)} + \text{操作数 (operand)}$$
MIPS 每条指令只做一次运算，复杂表达式需拆分：

高级语言 拆分 MIPS 指令
a = b + c - d; t = b + c add $t0, $s1, $s2
a = t - d sub $s0, $t0, $s3

其中 $t0 用作临时寄存器保存中间结果。

3.2 存储器

  • 32 位地址，4 GB 地址空间；32 位数据字长
  • 字节寻址：每个字节有一个独立地址
  • 每个字（Word）= 4 字节，字地址都是 4 的倍数

Example▶

字节寻址：

字地址 数据（十六进制） 说明
0x0000000C 40 F3 07 88 Word 3
0x00000008 01 EE 28 42 Word 2
0x00000004 F2 F1 AC 07 Word 1
0x00000000 AB CD EF 78 Word 0
每个字地址的宽度 = 4 字节，地址都是 4 的倍数。

字寻址：

字地址 数据（十六进制） 说明
0x00000003 40 F3 07 88 Word 3
0x00000002 01 EE 28 42 Word 2
0x00000001 F2 F1 AC 07 Word 1
0x00000000 AB CD EF 78 Word 0

字寻址 vs 字节寻址： 

> # 字寻址（以字为单位，偏移量 × 4）
> lw $s3, 1($0)     # 加载 Word 1
> sw $s7, 5($0)     # 存储到 Word 5
>
> # 字节寻址（以字节为单位，实际 MIPS 使用）
> lw $s3, 4($0)     # 加载地址 0x04 处的字
> sw $s7, 0x10($0)  # 存储到地址 0x10 处
>

3.3 立即数/常数（Immediate）

  • 因常数的值可以立即访问，故称为立即数（immediate）
  • 采用 16 位补码表示，范围 $[-32768, 32767]$
  • 减法相当于加上一个负数，故没有 subi 指令

# $s0 = a, $s1 = b
addi $s0, $s1, 4    # a = b + 4
addi $s1, $s0, -2   # b = a - 2

后缀i就是Immediate的缩写

4. MIPS 指令格式

所有指令均为 32 位，共有 3 种格式：

4.1 R 型（Register）

3个寄存器之间的操作，如 add、sub、sll、jr

op (6) rs (5) rt (5) rd (5) shamt (5) funct (6)
操作码 = 0 源寄存器 1 源寄存器 2 目的寄存器 移位量 函数码（决定 R 操作类型）

例：add $s0, $s1, $s2 的编码▶
字段 op rs rt rd shamt funct
值 0 17 18 16 0 32
二进制 000000 10001 10010 10000 00000 100000

机器指令：0x02328020

4.2 I 型（Immediate）

2 寄存器 + 16 位立即数，如 addi、lw、sw、beq、bne

op (6) rs (5) rt (5) imm (16)
操作码 源寄存器 目的寄存器 16 位立即数
  • 正立即数：高 16 位补 0
  • 负立即数：高 16 位补 1（符号扩展）

例：addi $s0, $s1, -4 的编码▶
字段 op rs rt imm
值 8 17 16 -4
二进制 001000 10001 10000 1111111111111100

机器指令：0x2232FFFC

4.3 J 型（Jump）

26 位地址操作数，如 j、jal

op (6) addr (26)
操作码 26 位地址

目标地址计算：$PC' = \{(PC+4)[31:28],\ addr,\ 2'b0\}$

将 32 位地址去掉前 4 位和后 2 位，保留中间 26 位。

5. MIPS 常用指令

5.1 算术指令

add  $s0, $s1, $s2    # $s0 = $s1 + $s2
sub  $s0, $s1, $s2    # $s0 = $s1 - $s2
addi $s0, $s1, 4      # $s0 = $s1 + 4

5.2 逻辑指令

R 型（寄存器操作）：

and  $s3, $s1, $s2    # 按位与
or   $s4, $s1, $s2    # 按位或
xor  $s5, $s1, $s2    # 按位异或
nor  $s6, $s1, $s2    # 按位或非

I 型（立即数操作，零扩展）：

andi $s2, $s1, 0xFA34
ori  $s3, $s1, 0xFA34
xori $s4, $s1, 0xFA34

没有 NOT 指令

可用 NOR 替代：A NOR $0 = NOT A

I 型逻辑指令的立即数是零扩展，而非符号扩展

5.3 移位指令

固定移位（shamt 编码在指令中）：

sll $t0, $s1, 2    # 逻辑左移：$t0 = $s1 << 2（低位补 0）
srl $s2, $s1, 2    # 逻辑右移：$s2 = $s1 >> 2（高位补 0）
sra $s3, $s1, 2    # 算术右移：$s3 = $s1 >>> 2（高位补符号位）

可变移位（移位量由寄存器给出）：

sllv $t0, $s1, $s2    # 可变逻辑左移（$s2 低 5 位为移位量）
srlv $t0, $s1, $s2    # 可变逻辑右移
srav $t0, $s1, $s2    # 可变算术右移

5.4 生成常数指令

16 位常数：直接用 addi

addi $s0, $0, 0x4F3C    # $s0 = 0x4F3C

32 位常数：lui（Load Upper Immediate）+ ori

lui $s0, 0xFEDC          # $s0 高 16 位 = 0xFEDC，低 16 位 = 0
ori $s0, $s0, 0x8765     # $s0 低 16 位 = 0x8765
# 最终 $s0 = 0xFEDC8765

5.5 乘法与除法指令

乘法和除法结果存储在专用的 HI/LO 寄存器中：

mult $s0, $s1    # {HI, LO} = $s0 × $s1
div  $s0, $s1    # LO = $s0 / $s1（商），HI = $s0 % $s1（余数）

5.6 数据传送指令

lw  $s0, 8($0)     # $s0 = Memory[$0 + 8]（从内存加载字）
sw  $s0, 8($0)     # Memory[$0 + 8] = $s0（向内存存储字）

5.7 条件分支指令

beq $s0, $s1, label    # if ($s0 == $s1) 跳转到 label
bne $s0, $s1, label    # if ($s0 != $s1) 跳转到 label

目标地址计算：$PC' = (PC + 4) + (SignImm \ll 2)$

5.8 无条件跳转指令

j   label    # 无条件跳转到 label
jal label    # 跳转并链接（$ra = PC + 4，然后跳转）
jr  $ra      # 跳转到寄存器保存的地址（常用于函数返回）

5.9 比较指令（Set Less Than）

slt $t1, $s0, $t0    # if ($s0 < $t0) $t1 = 1, else $t1 = 0

6. 程序控制结构

6.1 if-else 语句

if (i == j)
    f = g + h;
else
    f = f - i;

    bne $s3, $s4, else    # if (i != j) 跳转到 else
    add $s0, $s1, $s2     # f = g + h
    j   done              # 跳过 else
else:
    sub $s0, $s0, $s3     # f = f - i
done:

条件反转技巧

MIPS 中常把 C 语言的条件取反来写分支，即 if (i == j) 用 bne 实现。

6.2 while 循环

int pow = 1, x = 0;
while (pow != 128) {
    pow = pow * 2;
    x = x + 1;
}

      addi $s0, $0, 1       # pow = 1
      add  $s1, $0, $0      # x = 0
      addi $t0, $0, 128     # t0 = 128
while:
      beq  $s0, $t0, done   # if (pow == 128) exit
      sll  $s0, $s0, 1      # pow = pow * 2
      addi $s1, $s1, 1      # x = x + 1
      j    while
done:

6.3 for 循环

int sum = 0;
for (int i = 0; i != 10; i++)
    sum = sum + i;

      addi $s1, $0, 0       # sum = 0
      add  $s0, $0, $0      # i = 0
      addi $t0, $0, 10      # $t0 = 10
for:
      beq  $s0, $t0, done   # if (i == 10) exit
      add  $s1, $s1, $s0    # sum = sum + i
      addi $s0, $s0, 1      # i++
      j    for
done:

7. MIPS 寻址方式

寻址方式 描述 示例
寄存器寻址 操作数在寄存器中 add rd, rs, rt
立即数寻址 操作数为 16 位立即数 addi rt, rs, imm
基地址寻址 地址 = 基址寄存器 + 偏移量 lw rt, imm(rs)
PC 相对寻址 $PC' = (PC+4) + SignImm \times 4$ beq rs, rt, label
伪直接寻址 $PC' = \{(PC+4)[31:28],\ addr,\ 2'b0\}$ j label

8. 完整示例：高级语言 → 汇编 → 机器码

int f, g, y;  // 全局变量
int main(void) {
    f = 2; g = 3;
    y = sum(f, g);
    return y;
}
int sum(int a, int b) {
    return (a + b);
}

.data
f:
g:
y:

.text
main:
    addi $sp, $sp, -4     # 分配栈帧
    sw   $ra, 0($sp)      # 保存返回地址
    addi $a0, $0, 2       # $a0 = 2
    sw   $a0, f            # f = 2
    addi $a1, $0, 3       # $a1 = 3
    sw   $a1, g            # g = 3
    jal  sum               # 调用 sum
    sw   $v0, y            # y = sum()
    lw   $ra, 0($sp)      # 恢复返回地址
    addi $sp, $sp, 4      # 恢复栈指针
    jr   $ra               # 返回
sum:
    add  $v0, $a0, $a1    # $v0 = a + b
    jr   $ra               # 返回

9. MIPS 内存布局

地址范围 段 说明
0xFFFFFFFF ~ 0x80000000 保留段（Reserved） 操作系统使用
0x7FFFFFFC ↓ 栈段（Stack） $sp = 0x7FFFFFFC，向下生长
↑ 堆段（Heap） 动态数据，向上生长
0x10010000 堆起始
0x10008000 $gp（全局指针） 全局数据段中心，64 KB
0x10000000 全局数据段 全局/静态变量
0x00400000 代码段（Text） 程序指令，256 MB
0x00000000 保留 操作系统使用

10. QtSpim 仿真器

10.1 简介

QtSpim（简称 Spim）是一个支持 32 位 MIPS 指令集的MIPS 微处理器模拟器，可以直接打开 MIPS 汇编指令源程序（.asm 文件）。

主要界面包含：
- 寄存器窗口：查看所有寄存器的值
- 代码窗口：查看地址、机器码、汇编指令、反汇编代码
- 数据窗口：查看内存映像
- 消息窗口：显示 QtSpim 的输出信息

10.2 Hello World 示例

.data
str: .asciiz "Hello world.\n"

.text
.globl main
main:
    la   $a0, str      # 将字符串地址放入 $a0
    li   $v0, 4        # 系统调用号 4 = 打印字符串
    syscall             # 执行系统调用
    li   $v0, 10       # 系统调用号 10 = 退出程序
    syscall             # 退出

常见错误
  • 必须定义 main 标号（不能用其他名称如 top）
  • 引号必须使用半角双引号，不能用中文全角引号
  • 文件夹名称不能用中文
  • 汉字在 QtSpim 中显示为乱码

10.4 查看内存映像

  • 数据段起始地址：0x10010000
  • 数据在内存中按小端序（Little-endian）存放

字节序说明▶

MIPS 微处理器原本采用大端序（Big-endian），但由于 QtSpim 运行在 PC（Intel 处理器）上，实际数据存储采用小端序（Little-endian）。

AI 知识概要

核心主题

本实验围绕 MIPS 汇编程序设计展开，涵盖 MIPS 体系结构基础、指令格式、常用指令以及 QtSpim 仿真器的使用方法。

关键概念

  • CISC vs RISC：MIPS 属于 RISC 架构，指令精简、固定长度、只用 Load/Store 访问内存
  • 设计准则：简单规整化、加快常见功能、越小越快、好的折中
  • 32 个通用寄存器：$t0-$t9 临时变量、$s0-$s7 保存变量、$a0-$a3 函数参数、$v0-$v1 返回值
  • 三种指令格式：R 型（寄存器间操作）、I 型（含 16 位立即数）、J 型（26 位跳转地址）
  • 五种寻址方式：寄存器、立即数、基地址、PC 相对、伪直接寻址

重点指令

  • 算术：add、sub、addi（无 subi）
  • 逻辑：and/or/xor/nor（注意：无 NOT，用 NOR $0 替代）
  • 移位：sll/srl/sra（固定）、sllv/srlv/srav（可变）
  • 数据传送：lw/sw（Load/Store）
  • 分支跳转：beq/bne、j/jal/jr
  • 常数生成：lui + ori 构造 32 位常数

编程技巧

  • 条件反转：C 语言 if (==) 在 MIPS 中常用 bne 实现
  • 循环结构：while/for 均用 beq/bne + j 组合实现
  • 函数调用：jal 保存返回地址到 $ra，jr $ra 返回
  • 栈操作：$sp 管理栈帧，调用前保存 $ra

实验工具
